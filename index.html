Blog Post 4

The blog about abstraction and loops is really interesting because it shows that just by looping something, simplification isn't necessarily taking place. Abstraction is more based on grouping more specific commands into a general code with variables, like in a function. A loop is similar, but not quite the same; it doesn't have inputs and it doesn't really recurse (although this is arguable). Loops aren't abstracting code, theyre just recognizing that the same code may have to be written multiple times and using a more general code to simplify slightly.

A loop is not an example of abstraction. Loops make the code more effiecient by not repeating the same code over and over again but it isn't hidden from the coder. If a loop was in a function then it would be an abstraction but only because functions are an example of abstraction. If the loop uses a function then the function inside of the loop is an abstraction but not the loop itself. A loop is just a way to take repetitive code and put it all in one place.
